# Feetech STS3215 bus servo SDK. The `feetech-servo-sdk` PyPI package
# (import name `scservo_sdk`) turned out to only expose a low-level
# Dynamixel-SDK-style API (PortHandler + PacketHandler(protocol_end) with
# read/write<N>ByteTxRx(port, id, address, ...)) -- there's no high-level
# sms_sts-style convenience class, despite that being the initial
# assumption. arm_hardware.py's Servos talks the STS3215's register table
# directly against this low-level API; if you're on a different SDK fork
# that *does* expose a higher-level class, arm_hardware.py is the one file
# to adjust (see its module docstring). Confirm what your install exposes:
#   python -c "import scservo_sdk as s; print([n for n in dir(s) if not n.startswith('_')])"
feetech-servo-sdk
